About me

I’m a Computer Science & Engineering student at Texas A&M. I like building projects where usability and accessibility are treated as core requirements, not “nice-to-have” features.

What I’m looking for in teammates

I’m down to team up with people who care about accessibility testing, inclusive UI design, and practical features like captions, keyboard navigation, and readable layouts.

Accessibility interests

Topics I’m especially interested in:

  • WCAG-friendly UI (contrast, headings, focus states, semantics)
  • Assistive tech compatibility (screen readers, keyboard-only navigation)
  • Accessible media (captions, transcripts, clear audio)
  • Inclusive forms (labels, errors, helpful instructions)

Ways I can contribute

  1. Build semantic page structure with accessible components
  2. Run quick accessibility checks (keyboard pass + basic screen reader pass)
  3. Write clear alt text and ensure headings/landmarks make sense
